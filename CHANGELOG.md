# modiq 변경 로그

## v1.1.0 — 2026.06.22 · MVP 완료(정산·세무 서류 + 성능/UX) 🎉

v1.0.0(MVP) 이후 누적 개선을 모은 마무리 릴리스. 이후 개발은 랜딩 페이지·가격 정책으로 이동. 버전 표기 `1.1.0`으로 통일(package.json · constants.ts · README · 앱 내 표시).

### 정산 · 세무 서류
- **정산 발급 서류 세무유형 분기**: 프리랜서·외국인 = **원천징수 내역서**, 소속사(대대행) = **거래명세서**(공급가액+부가세 10%+합계, 세금계산서는 소속 에이전시 발행). 소속사는 원천징수 대상 아님 → 원천징수세액 0. (`SettlementStatementModal`, `lib/withholdingStatement.ts`)
- **원천징수 내역서 식별번호 라벨 자동화**: 내국인=주민등록번호 / 외국인=여권번호(외국인등록증 보유 시 외국인등록번호). `national_id_type` 기준 (`modelIdLabel`).
- **매출 거래명세서 / 청구 신설**(`ClientStatementModal`, `lib/clientStatement.ts`): 매출 화면 `📑 거래명세서 / 청구` 버튼 → 고객사 대상(공급자=당사) 거래명세서 발급.
  - 발급 단위 토글: **프로젝트별** / **고객사 월합산(한 장)**
  - 월별 리스트: 완납/부분입금/미수 칩 + 총청구·입금완료·미수금 합계 + `미수만 보기` 필터
  - 미리보기 → 인쇄/PDF → 이메일(고객사 `tax_email` 우선) → 엑셀 다운로드

### 모델 · 세무 식별정보
- **외국인 비자·정산 정보 모달 통합**: 외국인은 비자 유형·입출국·세율·지급방식 + **주소·외국인등록번호/여권번호**까지 한 화면(모달)에서 입력. 내국인 주민등록번호는 모델 폼 '세무 신고용 정보'로 분리. 둘 다 `canViewFinance`(대표·정산권한자) 전용.
- **세무 식별번호 암호화 저장**: `set_model_national_id`/`get_model_national_id` RPC로 평문 미노출(마스킹 표시), 복호화 접근은 감사 로그(`secure_id_access_log`) 기록.
- **생년월일 6자리 라벨 통일**(YYMMDD): 세무 주민번호 입력 시 생년월일 자동 대치.

### 표기 · UI
- **3사이즈(가슴·허리·엉덩이) inch 표기 통일**: 포트폴리오·패키지·컴카드 모두 inch. (저장은 cm 유지, cm 입력 → inch 자동 변환)
- 전체 폰트 확대(웹 +2pt / 모바일 +1pt), 다수 모바일 화면맞춤(캘린더·섭외상세·매출순위 등), 대시보드/매출 상단 숫자 일반체.
- 컴카드 리뉴얼(모달 포털·다운로드 클론·중앙 정보 정렬·닫기 버튼 위치), 패키지 컴카드 새 레이아웃 반영, 패키지↔포트폴리오 사진을 **갤러리 순서 기준**으로 1:1 동기화.

### 성능 · 사진
- 썸네일을 **Storage URL**로 전환 + 목록 가상화(증분 렌더)로 1000명 규모 대응.
- 사진 저장 파이프라인 정리: 원본 긴 변 **1500px / 품질 0.8**, 갤러리 썸네일 **480px**(공유 화면 선명도), 목록/아바타 썸네일 **150px**. 레거시 base64·썸네일 없는 사진은 포트폴리오 "썸네일 생성"·"저장소로 이전"으로 1회 정리.
