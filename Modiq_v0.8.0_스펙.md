# Modiq v0.8.0 버전업 스펙

> 작성일: 2026-06-10 · 기준 버전: 0.7.6 → **0.8.0**
> 범위: 신규 문의 인테이크 · 알림톡(카카오/Solapi) · 담당자 연락처 연결

---

## 1. 요약

v0.8.0은 "리드(문의)를 안 흘리고, 일정 변동을 자동으로 알리는" MVP 마무리 버전이다. 신규 기능은 두 덩어리다.

1. **대시보드 신규 문의 카드** — 미처리 문의(INQUIRY)를 대시보드 상단에 모아 보여주고, 오늘 들어온 건은 NEW 배지, 묵은 건은 경과일로 강조한다.
2. **카카오 알림톡 4종** — 확정 / 취소 / 전날 리마인드 / 변경. 발신번호 **010-8796-7966**, Solapi 발송.

## 2. 이미 구현되어 있던 것 (재확인)

설계 중 코드 점검 결과, 아래는 **이미 존재**하므로 새로 만들지 않는다.

- `INQUIRY`(문의접수) 상태 — `constants.ts`의 `STATUS`에 정의됨. 섭외 추가 시 상태로 선택 가능.
- **담당자 = 멤버 선택 드롭다운** — 섭외 추가/수정 폼 모두 `agency_members`를 `<select>`로 노출(이름 저장). 별도 입력란 불필요.
- `agency_members.phone` 필드 — 멤버 등록 화면에 연락처 입력칸 존재. (채워 넣기만 하면 알림톡 담당자 연락처로 사용 가능)

→ 따라서 "에이전시 정보 입력란 추가"는 **대표번호(fallback) 1칸**만 선택 적용하면 충분하다(아래 4-3).

## 3. 신규 구현 — 대시보드 신규 문의 카드

**위치:** `src/views/DashboardView.tsx` (통계 카드 4개 아래, 이번 주 섭외 위)

**규칙(중요):** 카드 기준은 "오늘"이 아니라 **미처리 상태(`status==="INQUIRY"`)**다. 날짜 기준으로 하면 자정 지나면서 어제 문의가 사라져 리드를 놓친다.

- 카드 = INQUIRY 상태 전체 건수("처리 대기"), 헤더에 "오늘 N건" 부기
- 각 행: 유형 칩 · 모델(미정 허용) · 고객사(미정 허용) · 희망일(미정 허용)
- `created_at`이 오늘이면 **NEW 배지**, 1일 이상 지났으면 **N일 경과**(3일↑ 빨강)
- 행 클릭 → 기존 섭외 상세 모달(`setSelectedBooking`) → 거기서 상태를 확정(전환)/취소(거절)로 변경
- INQUIRY는 "진행중 섭외" 통계/목록에서 제외(중복 표시 방지, 의미상 raw 문의는 진행중이 아님)

**인테이크 방식:** 공개 폼 없음. 카카오/채팅으로 들어온 문의를 매니저가 섭외 추가(상태=문의접수)로 수동 등록 → 카드에 자동 노출. (공개 요청폼은 문의량이 많아지면 v1.1)

## 4. 신규 구현 — 카카오 알림톡 4종

### 4-1. 템플릿 (카카오 사전 승인 필요)

`#{유형}` = 촬영 / 실물미팅 / 피팅 / 오디션. booking_type만 매핑하면 4개로 전 유형 처리.

| 코드 | 트리거 | 수신자(권장) |
|---|---|---|
| `MODIQ_CONFIRM_01` | 상태 → 확정 | 모델 (+고객사) |
| `MODIQ_CANCEL_01` | 상태 → 취소 | 모델 (+고객사) |
| `MODIQ_REMIND_01` | 촬영 전날(스케줄러) | 모델 |
| `MODIQ_CHANGE_01` | 일시/장소 변경 | 모델 (+고객사) |

본문 원문은 `src/lib/alimtalk.ts`의 `buildAlimtalkText`에 1:1로 들어 있다. 카카오 템플릿 등록 시 이 문구를 그대로 쓰되 값 부분만 `#{변수}`로 치환해 등록한다.

### 4-2. 발송 경로 (보안)

프론트에서 Solapi API 키를 직접 들고 있으면 안 된다. 발송은 **Supabase Edge Function(`solapi-send`)** 경유.

```
프론트 sendAlimtalk()  →  POST {VITE_SOLAPI_FN_URL}  →  Edge Function(키 보관)  →  Solapi
```

`src/lib/alimtalk.ts`의 `sendAlimtalk`는 `VITE_SOLAPI_FN_URL` 미설정 시 **콘솔 로그만 남기고 no-op**(절대 throw 안 함). 즉 엣지 함수 배포 전에도 앱은 안전하게 돌아간다.

### 4-3. 데이터 / SQL

```sql
-- 담당자 미지정 시 fallback 대표번호 (선택)
ALTER TABLE agencies ADD COLUMN IF NOT EXISTS rep_phone TEXT DEFAULT '';
```

담당자 연락처 우선순위: `agency_members.phone`(매니저) → `agencies.rep_phone`(대표) → 없음.

### 4-4. 트리거 연결 지점 (다음 단계, App.tsx)

> 엣지 함수 배포 + 카카오 템플릿 승인 후 연결한다. 승인 전 연결해도 no-op라 무해하지만, 순서상 외부 준비를 먼저 끝낸다.

- **확정/취소:** `handleSaveBooking`/상태 변경 지점에서 status가 CONFIRMED/CANCELLED로 바뀔 때 `sendAlimtalk(모델폰, "CONFIRM"|"CANCEL", ...)`
- **변경:** 저장 시 기존 일시/장소와 비교해 달라지면 `sendAlimtalk(모델폰, "CHANGE", {before, after})`
- **리마인드:** 매일 1회 스케줄러(엣지 함수 Cron)에서 내일 날짜 섭외 조회 → `REMIND` 발송 (프론트 아님)

## 5. 외부 준비 단계 (코드로 못 하는 것)

1. **카카오 비즈니스 채널** 개설 + 사업자 정보 등록

4. 프론트 환경변수 `VITE_SOLAPI_FN_URL`에 엣지 함수 URL 설정
