# Modiq Q&A (v1.2.0)

운영 중 자주 나오는 질문 모음. (개발 세부는 [DEVELOPMENT.md](DEVELOPMENT.md), 사용법은 [MANUAL.md](MANUAL.md))

---

## 계정 · 권한

**Q. 대표 권한을 다른 사람에게 주고 싶어요.**
A. 설정 → 담당자에서 **공동 대표 권한 부여**가 가능합니다(소유권 이전이 아니라 권한 추가). 둘 다 owner가 됩니다.

**Q. 매니저가 매출·정산을 못 봐요.**
A. 재무 열람은 기본적으로 대표만 가능합니다. 담당자 설정에서 해당 멤버에게 재무 열람 권한을 별도 부여하세요.

**Q. 다른 PC(회사 등)에서 이어서 작업하려면?**
A. 데이터·시크릿은 클라우드(Supabase)라 PC와 무관하게 동일합니다. 코드 작업 폴더만 git으로 Push/Pull 하면 됩니다.

## 모델 · 사진

**Q. 모델을 클릭하면 오른쪽 갤러리가 느리게 떠요.**
A. v1.0.0에서 목록 썸네일을 작은 이미지로 받도록 개선했습니다. 추가로 포트폴리오 우상단 **"썸네일 생성"** 버튼을 1회 실행하면 기존 사진의 작은 썸네일이 만들어져 가장 빨라집니다. base64로 저장된 옛 사진은 **"저장소로 이전"** 도 함께 실행하세요.

**Q. 사진은 몇 장까지 올릴 수 있나요?**
A. 모델당 **최대 30장**. 저장 시 자동으로 **긴 변 1500px(품질 0.8)** 로 정리되고, 갤러리용 480px·목록용 150px 썸네일이 함께 만들어져 용량/로딩 걱정이 없습니다.

**Q. 3사이즈(가슴·허리·엉덩이)는 cm인가요 inch인가요?**
A. **표기는 inch로 통일**(포트폴리오·패키지·컴카드 동일)입니다. 입력은 inch 우선이고, cm로 넣으면 inch로 자동 변환되어 표시됩니다(내부 저장은 cm 유지).

**Q. 대표 사진은 어떻게 정하나요?**
A. 갤러리 사진의 "대표 지정" 버튼을 누르거나, 프로필의 "대표 사진"으로 직접 업로드합니다. 목록·카드·컴카드의 썸네일로 쓰입니다.

**Q. 검색 필터에서 어떤 모델이 안 보여요.**
A. 나이/신장/신발/모델료 같은 **범위 필터는 값이 입력된 모델만** 거릅니다(값이 비면 통과). 단 성별·분야 같은 칩 필터는 해당 값이 있어야 잡힙니다. 해당 항목이 비어 있으면 입력 후 다시 검색하세요.

**Q. 외국인 모델은 뭐가 다른가요? 비자정보는 어디서 입력하나요?**
A. 모델 등록 화면의 "외국인 모델" 토글을 켜면 **[비자·정산 정보]** 모달이 열립니다. 여기서 국적·비자타입·입출국일·외국인등록증·세율·지급방식 **그리고 주소·외국인등록번호/여권번호까지 한 번에** 입력합니다. 정산 세무유형(E-6 3.3% / C-4·기타 20%)에 자동 반영됩니다.

**Q. 주민번호(세무 식별번호) 입력칸이 안 보여요.**
A. 식별번호·주소가 있는 **'세무 신고용 정보'(내국인) / [비자·정산 정보](외국인)** 는 **대표·정산권한자(재무 열람)** 계정에만 보입니다. 권한 없는 멤버 계정에서는 숨겨지는 것이 **정상**입니다(개인정보 보호). 식별번호는 암호화 저장되어 화면엔 마스킹만 표시됩니다.

**Q. 생년월일 6자리와 세무 주민번호는 무슨 관계인가요?**
A. 상단 **생년월일 6자리(YYMMDD)** 는 나이 표시·ID 생성용입니다. '세무 신고용 정보'에 **주민등록번호**를 입력하면 그 앞 6자리가 생년월일 칸에 자동으로 채워집니다.

## 섭외 · 상태

**Q. 카카오/전화로 들어온 문의는 어떻게 관리하나요?**
A. 섭외 추가 시 상태를 **문의접수**로 등록하세요. 대시보드 "신규 문의" 카드에 모이고, 행을 클릭해 확정/취소로 전환합니다. (공개 섭외 요청폼은 로드맵)

**Q. 신규 문의 카드가 자정 지나면 사라지나요?**
A. 아니요. 카드 기준은 날짜가 아니라 **미처리 상태(문의접수)** 입니다. 처리(확정/취소)하기 전까지 계속 보입니다.

**Q. 미팅·피팅·오디션도 촬영처럼 상태가 복잡한가요?**
A. 아니요. 비촬영 타입은 **요청 → 확정 → 완료**(+취소) 3단계로 단순화되어 표시됩니다.

**Q. 모델 휴무와 겹치는 일정을 잡으면?**
A. 추가·수정 시 **충돌 경고**가 뜨지만, 필요하면 강행해서 등록할 수 있습니다. 캘린더에는 휴무가 음영/"휴무 N"으로 표시됩니다.

## 정산 · 매출

**Q. 모델 정산액은 어떻게 계산되나요?**
A. 섭외 시간으로 세션을 판정(≤5h Half, 6h↑ Day)한 뒤 모델료에 정산방식을 적용합니다. **비율**이면 모델료×%, **정액**이면 입력 금액 그대로입니다.

**Q. "매출총이익"은 무엇을 뺀 값인가요?**
A. **매출(공급가) − 모델 정산 기준액** 입니다. 모델 지급액·세금이 아니라 정산 기준액을 차감합니다. 월 고정비를 반영한 **영업이익은 아직 미구현**(로드맵)입니다.

**Q. 정산/매출을 엑셀로 받을 수 있나요?**
A. 매출 현황에서 **.xlsx 다운로드**가 가능합니다(섭외ID·계약금·잔금·모델지급·총이익·합계 포함).

**Q. 고객사 입금 여부는 어디서 보나요?**
A. 매출 건별 내역에 입금/미입금이 읽기전용으로 표시됩니다. 더 자세히는 매출 화면의 **`📑 거래명세서 / 청구`** 에서 프로젝트/고객사별 **완납·부분입금·미수**와 **미수금 합계**를 한눈에 보고 `미수만 보기`로 거를 수 있습니다.

**Q. 소속사(대대행) 모델은 원천징수 내역서가 맞나요?**
A. 아니요. 소속사 모델은 **원천징수 대상이 아니라** 소속 에이전시가 세금계산서를 발행합니다. 그래서 정산 내역서에서 소속사 모델을 선택하면 **거래명세서**(공급가액+부가세 10%+합계)가 발급됩니다. 프리랜서·외국인만 **원천징수 내역서**가 나옵니다.

**Q. 고객사에 보내는 거래명세서(청구서)는 어떻게 발급하나요?**
A. 매출 화면 → **`📑 거래명세서 / 청구`**. **프로젝트별** 또는 **고객사 월합산(한 장)** 으로 발급 단위를 고르고, 행의 **`🧾 명세서 발급`** 으로 미리보기 → 인쇄/PDF → 이메일 발송(고객사 '계산서 발송 이메일' 우선)합니다. 실제 세금계산서는 홈택스에서 별도 발행하세요.

**Q. 한 달치를 한 장으로 청구할 수 있나요?**
A. 네. `📑 거래명세서 / 청구`에서 기간을 **이번 달**로 두고 **고객사 월합산** 모드를 선택하면, 그 고객사의 한 달 전체 거래가 한 장의 거래명세서로 묶여 발급됩니다.

## 캘린더 · 알림

**Q. 섭외를 확정했는데 캘린더에 일정이 안 생겨요. (v1.2.0)**
A. v1.2.0부터 **모델이 수락해야** 캘린더에 일정이 생깁니다. 확정하면 모델에게 **[수락]/[거절] 초대 메일**이 가고, 모델이 [수락]을 누르는 순간 에이전시 구글 캘린더에 생성됩니다. 모델 응답은 섭외 상세의 **배지**(수락 대기/수락됨/거절)로 확인하세요. 초대 메일은 **Resend 도메인 인증**이 되어 있어야 실제 발송됩니다. 수락을 기다리지 않고 즉시 넣으려면 **"일정 보내기 → 구글 캘린더 등록"**(수동)을 쓰세요.

**Q. 모델이 메일에 회신하면 어디로 가나요?**
A. 발신자는 **"modiq"** 로 표시되지만, **회신(Reply-To)은 담당 에이전시 이메일**로 설정돼 있어 모델이 답장하면 에이전시가 받습니다. (발신 도메인 `@modiq.kr`은 공용이고, 에이전시 정체성은 제목·본문·회신주소로 전달)

**Q. 모델이 구글 계정이 아니어도 일정이 동기화되나요?**
A. 네. 수락은 메일의 버튼(웹페이지)으로 처리돼 **어떤 이메일이든** 됩니다. 수락 후 페이지에서 **.ics 파일**(애플·아웃룩·네이버 등) 또는 **구글 캘린더에 추가** 버튼으로 자기 캘린더에 넣을 수 있습니다.

**Q. 모델이 안드로이드인데 캘린더 구독이 안 돼요.**
A. 안드로이드 구글 캘린더 앱은 URL 구독을 직접 받지 못합니다. **ICSx⁵**(무료) 앱을 설치하면 구독 링크로 바로 동기화됩니다. 구독 링크 페이지가 기종을 자동 감지해 설치/구독 버튼을 안내합니다.

**Q. 일정을 바꿨는데 모델 폰에 바로 안 떠요.**
A. 구독형은 **주기적 동기화**(실시간 아님)입니다. 애플·ICSx⁵는 갱신 주기를 짧게 설정할 수 있고, 구글 웹은 느립니다. **급한 변경·취소는 알림톡/이메일로 별도 통보**하세요.

**Q. 알림톡이 안 나가요.**
A. 알림톡은 외부 준비가 끝나야 동작합니다: 카카오 비즈니스 채널 + 템플릿 4종 승인 → `solapi-send` Edge Function 배포 → `VITE_SOLAPI_FN_URL` 설정. 준비 전에는 안전하게 무시(no-op)됩니다. 담당자 연락처(멤버 phone 또는 회사 rep_phone)도 채워져 있어야 합니다.

## 데이터 · 운영

**Q. "컬럼이 없다"는 에러가 떠요.**
A. 해당 기능의 마이그레이션 SQL이 아직 실행되지 않은 것입니다. `supabase/` 폴더의 관련 `*_setup.sql`을 Supabase SQL Editor에서 실행하세요(예: 모델료 → `model_fee_setup.sql`).

**Q. 실고객 데이터를 넣기 전에 챙길 것은?**
A. **RLS(Row Level Security)** 정책 점검이 최우선입니다. 새로 추가한 테이블(`model_offs` 등)에도 에이전시 격리 정책이 적용됐는지 확인하고, Supabase 키를 `.env`로 분리하는 것을 권장합니다.

**Q. 코드가 날아가면 복원되나요?**
A. 프로젝트는 독립 폴더라 `npm install` → `npm run dev`로 어디서든 복원됩니다. `.versions/`에 원본 스냅샷 백업도 있습니다.

## 버전

**Q. 현재 버전과 다음 계획은?**
A. **v1.2.0** — 섭외 **수락형 일정 동기화**(확정→모델 수락→캘린더 생성), 메일 **발신=modiq·회신=에이전시** 정비. (이전 **v1.1.0**: 정산·세무 서류(원천징수/거래명세서), 매출 거래명세서·청구, 외국인 비자·정산 정보 통합, inch 표기, 성능/UX.) 기능 후보: 모델 수락률 집계, 영업이익(월 고정비) 단계, 체류 만료 알림, 공개 섭외 요청폼, 세금계산서 자동화(팝빌/바로빌).
