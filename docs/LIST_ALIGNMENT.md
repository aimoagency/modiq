# 리스트 정렬 규칙 (엑셀형 표) — 🔒 고정 규칙

> 모딕의 모든 목록(섭외·대시보드·매출·정산·고객사·담당자·패키지 등)은 **엑셀형 표**로 정렬한다.
> 아래는 반복된 정렬 이슈를 끝낸 합의 규칙이다. **임의로 되돌리지 말 것.** 레이아웃 변경 시 이 문서를 먼저 확인한다.

## 1. 기본 구조 (모든 목록 공통)
- **하나의 컨테이너** `div`(border/radius/overflow:hidden/background:C.card) 안에:
  1. (데스크탑) **헤더 행** — 구분 제목(컬럼명).
  2. **데이터 행들** — 각 행은 `display:grid`.
- 헤더 행과 데이터 행은 **같은 `GRID` 문자열**(상수)을 공유한다.
- 행 구분선은 **first-flag divider**: 첫 행은 `borderTop:none`, 이후 `1px solid C.border`.
- hover 시 `background:C.card2`.
- `gap`은 전 행 동일(보통 `14`).

## 2. ⚠️ 가장 중요 — "별개 grid 컨테이너" 함정 (재발 금지)
**헤더 행과 각 데이터 행은 서로 다른 CSS grid 컨테이너다.** 따라서 트랙 폭은 컨테이너마다 **독립적으로** 계산된다.
- `minmax(0, N fr)` 트랙 → 폭이 **내용과 무관**(남는 폭 비율로만 결정)하므로 컨테이너가 달라도 **동일**하게 계산된다. ✅ 정렬 일치.
- `max-content` / `auto` 트랙 → **내용 폭으로 결정**되므로 컨테이너마다 다르다. ❌
  - 예) 마지막 `상태` 컬럼을 `max-content`로 두면 헤더에선 "상태" 글자폭(≈28px), 데이터행에선 배지폭(≈70px)으로 계산 → 남는 폭이 달라져 **앞의 fr 트랙들이 어긋나고 금액·상태 제목이 셀과 안 맞는다.**

### 규칙 (반드시 준수)
> **헤더+데이터가 공유하는 GRID의 모든 트랙은 "결정적"이어야 한다.**
> 허용: `고정 px`(예: `56px`, `88px`), `minmax(0, N fr)`.
> 금지: 공유 GRID 안의 **벌거벗은 `max-content` / `auto`**.
- 금액처럼 내용폭 보장이 필요하면 `minmax(max-content, N fr)`를 쓴다(폭 보장 + 균등 분배, 잘림 방지). 단, 이는 **헤더에도 같은 GRID가 적용될 때만** 안전. 헤더 텍스트가 데이터보다 좁으면 데이터행 기준으로 트랙이 커져 정렬이 어긋날 수 있으니, **배지/짧은 토큰 컬럼은 고정 px**로 둔다.
- **섭외 리스트(`BookingsList`)**: `상태`=고정 `88px`(배지 폭). 나머지는 고정 px(유형 56·아바타 32) + `minmax(0,fr)`.

## 3. 섭외 리스트는 공용 컴포넌트 1개 (`src/components/BookingsList.tsx`)
- **섭외 화면 / 대시보드 "진행중 섭외" / 대시보드 "신규문의"** 는 모두 `BookingsList`를 쓴다.
  - 같은 코드 → 컬럼·정렬·헤더가 항상 100% 동일. 한 곳만 고치면 셋 다 반영.
- props: `bookings, models, customers, isMobile, onSelect, showHeader?, emptyText?`.
- 신규문의는 핑크 알림 헤더(건수·NEW)만 유지하고 내부 목록은 `showHeader={false}`로 이 컴포넌트를 쓴다.
- GRID(9컬럼): `56px 32px minmax(0,2fr) minmax(0,1.1fr) minmax(0,1.2fr) minmax(0,1.4fr) minmax(0,1fr) minmax(0,1.1fr) 88px`
  = 유형·아바타·모델→고객사·프로젝트·날짜·장소·담당자·금액·상태.

## 4. 헤더(구분 제목) 정렬
- 기본 원칙: **헤더 라벨 정렬 = 그 컬럼 데이터 정렬** → 제목이 셀 "바로 위"에 온다.
  - 좌측 텍스트 컬럼(모델→고객사·프로젝트 등) = 좌측.
  - 금액류 = 우측(`textAlign:right`). 배지(상태) = `display:flex; justifyContent:flex-end`(헤더·데이터 동일).
- **예외(사용자 지정)**: **정산 화면 헤더는 셀 중앙(센터) 정렬**(`textAlign:center` 헤더 래퍼). 데이터는 각 컬럼 고유 정렬 유지.

## 5. 빈칸 규칙
- 값이 없으면 **빈 슬롯을 그대로 유지**한다(`""` span 렌더). 셀을 합치거나 없애지 않는다.
  - 예) 단건 섭외의 모델수, 계약금 0 → 빈칸.

## 7. 금액(총이익 등) 잘림 금지
- 금액 컬럼은 우측 정렬 + `whiteSpace:nowrap`. 좌우 마진을 고려해 **절대 잘리지 않게** 한다.
- 잘릴 위험이 있으면 해당 컬럼을 `minmax(max-content, fr)`(또는 고정 px)로 폭을 보장한다.
