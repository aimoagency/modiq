## 1. 한 줄 정의
**모딕 = 탤런트(모델) 에이전시 운영 SaaS ("talent agency OS").** 모델 등록·포트폴리오·패키지 제안·섭외(스케줄)·캘린더·정산·세무·매출·대대행 발송을 한곳에서.

- 대상: 모델/탤런트 에이전시(1인 ~ 중대형), 외국인 모델 포함.
- 형태: 웹 SPA(데스크탑·모바일 반응형), 클라우드(설치 불필요).

| 메뉴 | 핵심 기능 | 권한 |
|---|---|---|
| **대시보드** | 이번 달 매출 요약, 입금 확인 필요, 진행중 섭외, 신규 문의, 이번 주 섭외, HOLD 경고 | 전체(재무 카드는 권한자) |
| **섭외** | 섭외(촬영·실물미팅·피팅·오디션) 등록·상태관리·필터, 프로젝트 단위 그룹, 스케줄 충돌/HOLD | 전체 |
| **모델(탤런트)** | 모델 등록·프로필·신체·비자·정산정보, 출처(대대행) 필터, **활동중/보관 상태** | 전체(세무 식별번호는 재무권한자) |
| **포트폴리오** | 모델 사진 갤러리 + **영상(유튜브·비메오·인스타·틱톡, 9:16)** | 전체 |
| **패키지** | 모델 묶음 제안서(캐스팅/컴카드) 제작 → 공유 링크·PDF, 영상 첨부 | 전체 |
| **고객사** | 고객사(거래처) 등록·사업자정보·담당자·세금계산서 이메일 | 전체 |
| **발송(대대행)** | A 업체가 모델을 B 업체에 발송 → B가 "내 모델로 등록"(소속사 편입) | Pro+ (요금제) |
| **매출 현황** | 기간별 매출·미수금·총이익, 고객사/모델 순위, 거래명세서·청구 | 재무권한자 |
| **정산** | 정산대기/정산완료/미입금잔금 탭, 모델 지급·고객 입금 토글, 원천징수/거래명세서 발급 | 재무권한자 |
| **담당자** | 멤버(좌석) 추가·권한(매출·정산 열람)·소유권 이전 | 대표 |
| **설정(회사정보)** | 상호·사업자·대표·주소·계좌, 구글 캘린더 연동, **비밀번호 변경**, 공동대표 권한 | 대표 |
| **요금제** | 플랜 비교·변경, 14일 체험 | 대표 |

## 4. 권한 모델
- **대표(owner)**: 전체. 담당자·회사설정·소유권 이전·요금제.
- **담당자(member)**: 기본은 매출·정산 숨김. 대표가 `can_view_finance` ON 시 재무 열람.
- `canViewFinance = owner || member.can_view_finance`. 세무 식별번호·재무 화면은 이 권한 전용.

## 5. 핵심 도메인 규칙 (요약 — 계산식·필터는 임의 변경 금지)
- **섭외 타입**: 촬영(SHOOT)·실물미팅(MEETING)·피팅·오디션. **계약형(hasContract)만 매출 인정**.
- **수락형 캘린더(v1.2)**: 확정 시 모델에게 [수락/거절] 초대 메일만 → 모델 수락 시 구글 캘린더 생성(에이전시 일정 + 모델 게스트). 렌더는 앱 정적 `public/respond.html`.
- **스케줄 충돌/HOLD(순서 의존)**: 시간이 실제 겹치거나 버퍼 미만일 때만. 먼저=촬영→다음 2시간 / 먼저=비촬영→촬영 1시간 / 비촬영끼리 같은 장소 면제·다른 장소 1시간. 충돌 시 **비촬영을 HOLD, 촬영 유지**.
- **정산 탭 분류**: 정산대기=촬영완료 & (고객입금·모델지급 둘 다 완료 아님) / 정산완료=둘 다 완료(또는 SETTLED) / 미입금잔금=고객 미입금.
- **정산 서류**: 프리랜서·외국인=원천징수 내역서 / 소속사(대대행)=거래명세서(공급가+VAT10%). 소속사 원천징수 0. (`modelPayout` 소속사=기준액×1.1, `modelWithholding`=0 — 정정 금지)
- **매출 인정**: 확정·완료·정산 상태 + 계약형 + 금액>0 + 촬영일이 기간 내. 미래월 건은 해당 월에서 보임(버그 아님).
- **세무 식별번호**: 내국인=모델 폼 주민번호 / 외국인=비자·정산 모달(주소+외국인등록번호/여권). 암호화 저장(RPC·감사로그), 재무권한자 전용.
- **대대행 V4**: B가 받은 모델 등록 시 **소속사(company) 10% 고정** 편입(프리/외국인 변경 불가). A 업체정보는 발송 스냅샷(`talent_distributions.sender_payout_info`)에서 자동 입력. 출처 자동 기록 + '대대행' 배지. 정산은 "A 지급액(○○ 대대행)" 표기(기존 company 경로 재사용).
- **모델 상태(v1.3)**: 활동중/보관 2단계. 보관=목록·섭외/패키지 선택·공개·대시보드 카운트에서 숨김, **이력 보존**(복귀 가능). 완전삭제는 이력 0건 & 대표만.

## 6. 요금제 (v3.0)
모든 표시가 **부가세(10%) 포함가**, 월요금, **14일 무료 체험** 후 결제.

| 플랜 | 월요금(VAT포함) | 담당자 | 한 줄 |
|---|---|---|---|
| **Starter** | ₩78,000 | 1명 | 1인 에이전시·대표 단독 |
| **Pro** | ₩118,000 | 3명 | 대표 + 직원 1~2명 |
| **Team (BEST)** | ₩158,000 | 7명 | 팀·외국인 모델 에이전시 |
| **Enterprise** | ₩228,000 | 15명(초과 +₩12,000/명) | 중대형·온보딩 교육 |

**기능 차등(요약)**
- 정산·세무: 핵심 3종(원천징수 자동계산·세무 엑셀·원천징수 내역서)=전 플랜. **업체별 정산 명세서=Pro부터**, **외국인 세무=Starter 제외·Pro +4만·Team부터 포함**.
- 커뮤니케이션: **이메일 알림(확정·취소·일정변동)·구글 캘린더 동기화=가능**. 알림톡·세금계산서 발행·이메일 대량발송=**예정**.
- 대대행: **Starter 발신·수신 불가, Pro부터 가능**.

## 7. 데이터 모델 (주요 테이블)
- `agencies` — 에이전시(테넌트): 상호·사업자·대표·주소·계좌, plan·trial_ends_at, payout_bank_info.
- `agency_members` — 담당자(좌석): role(owner/member)·can_view_finance.
- `models` — 모델/탤런트: 신체·비자·정산(payout_tax_type)·세무 식별번호(암호화)·videos(jsonb)·source_*(대대행 출처)·status(active/archived).
- `customers` — 고객사: 사업자정보·담당자·tax_email.
- `bookings` — 섭외: 타입·상태·일시·장소·금액·입금/지급 토글·project_id·model_response(수락형).
- `talent_distributions` / `distribution_models` — 발송(대대행) + 모델 스냅샷.
- `packages` — 패키지(items jsonb: 모델·사진·영상).

## 8. 백엔드 / 연동 (🔒 보호 영역)
- **Edge Functions**: `email-send`·`cal-feed`·`gcal-oauth`·`gcal-sync`·`booking-respond`·`solapi-send`.
- `lib/email.ts`·`lib/calendar.ts` = 동작하는 발송/동기화 로직 — 요청·확인 없이 수정 금지.
- ⚠️ 이메일 실패(502 등)는 대개 **코드가 아니라 Resend 도메인 인증/외부 설정** 문제. Edge Function은 앱과 분리 — 직접 재배포해야 반영.

## 9. 보호 영역 (재발 방지 · 임의 수정 금지)
- **대시보드 로딩 5기둥**(캐시 동기주입·loadData 일괄반영·스켈레톤 `&&` 가드·로그인 캐시복원·부팅 스플래시) → 0/흰화면 깜빡임 방지.
- **리스트(엑셀형) 정렬**: 공용 `BookingsList`로 섭외·대시보드·신규문의 통일. 헤더·데이터 공유 GRID의 모든 트랙은 **고정 px 또는 `minmax(0,fr)`**(벌거벗은 `max-content` 금지 — 헤더가 셀과 어긋나는 원인). 상세: [LIST_ALIGNMENT.md](LIST_ALIGNMENT.md).
- **이메일/캘린더 동기화** · **디자인**(승낙 후 변경) · **메뉴**(삭제 금지).

## 11. 로드맵
- **탤런트 시스템 확장**(검증용 프리뷰 `preview/talent-preview.html`): "탤런트" 아래 유형(모델·포토그래퍼·헤어·메이크업·스타일리스트·영상) = **출연(모델)/제작(크루)** 2군. 제작군은 프로젝트 섭외에 **비계약형(비용)** 배정, 정산은 프리/사업자 재사용, 패키지엔 **유형 라벨 카드**로 노출. 단계: 골격→섭외/정산→패키지.
- **예정 기능**: 알림톡(Solapi)·세금계산서 자동발행·이메일 대량발송·비밀번호 이메일 재설정 — 외부 인프라(Resend 도메인 인증 등) 완료 후.

## 12. 운영 메모
- ⚠️ 임시 컨테이너 작업은 **커밋·푸시까지** 완료해야 유실 없음.
- 배포 흐름: `claude/* → master`(머지 커밋). 어드민은 별도 수동 배포.
- 신규 기능 중 DB 컬럼 추가가 필요한 것(영상·발송 V4·모델 status)은 **Supabase SQL 적용 필요**(미적용 시 폴백으로 앱은 안전).
