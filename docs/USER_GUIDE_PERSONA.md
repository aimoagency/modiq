## 1. 김대표 — 오너/대표

> **한 줄 소개**: 회사를 만들고, 누가 무엇을 보고 할 수 있는지 정하며, 요금제·정산 최종 확인·소유권까지 책임지는 사람.

### 자주 쓰는 메뉴
**회사정보(설정)** · **담당자**(대표 전용) · **요금제** · **정산**(재무) · **매출 현황**(재무) · 대시보드.

### 첫 1시간 셋업 워크플로우
1. **회원가입 → 에이전시 생성**: 첫 사용자는 가입 후 에이전시를 만들면 자동으로 **대표(owner)** 가 됩니다.
2. **회사정보 입력**(설정/회사정보): 에이전시명, **대표 연락처(`rep_phone`)** — 알림톡 담당자 fallback로 쓰임, 사업자등록증(OCR 자동 인식).
   - 🤝 **대대행으로 모델을 보낼 계획이면 여기서 회사 정산정보까지 1회 등록**: 상호·사업자번호·대표·주소 + **정산 입금계좌(`agencies.payout_bank_info`)**. → [④-A](#a-a-업체-모델을-보내는-쪽) 참고. (이 정보가 발송 스냅샷으로 상대 업체에 자동 전달됩니다.)
3. **담당자 추가**(담당자 메뉴): 매니저를 등록하고 각자 **연락처(phone)** 를 채웁니다(알림톡 담당자 연락처로 사용).
4. **재무 권한 부여 결정**: 매니저별로 **`can_view_finance` 토글**을 켜야 그 사람이 매출·정산·세무 식별번호를 봅니다. 켜지 않으면 숨김(정상).
5. **요금제 선택**(요금제 메뉴): **Starter 78,000 / Pro 118,000 / Team 158,000(BEST) / Enterprise 228,000**(월·부가세 포함). 신규 가입은 **14일 무료체험(trial)** 으로 시작. 알림톡은 Standard 이상 기능.
6. (선택) 모델·고객사·섭외는 매니저에게 위임 → 김대표는 **대시보드 + 정산 승인**에 집중.

### 정산 "승인"(확인) 관점
- 모딕에 별도 승인 버튼이 있는 게 아니라, **정산 탭 분류로 진행 상황을 확인**합니다.
  - **정산대기** = 촬영완료 & (고객입금·모델지급 중 하나라도 미완).
  - **정산완료** = 고객입금 AND 모델지급 둘 다 완료(또는 상태 SETTLED).
  - **미입금잔금** = 고객 미입금.
- 대표는 정산 화면에서 **모델 발급 서류**(프리랜서·외국인=원천징수 내역서 / 소속사=거래명세서)와 **세무사용(주민번호 포함) 엑셀**을 받을 수 있습니다. 이 다운로드는 **대표·정산권한자만** 가능하며 **보안 감사 로그**가 남습니다.

### 소유권 이전 / 공동 대표
위치: **설정 → "권한 부여 (공동 대표)" 카드.**
- **권한 부여**: 담당자 선택 → `권한 부여` → 그 사람도 대표(owner)가 됨(본인 권한 유지).
- **권한 회수**: "현재 대표 권한 보유자"에서 이름 옆 **✕**. ⚠️ **본인 권한은 회수 불가**, **대표 최소 1명 유지**(마지막 대표 회수 불가).
- **완전 교체 순서**: ① 새 사람 담당자 추가 → 권한 부여 → ② 로그아웃 후 새 대표로 로그인 → ③ 기존 대표 옆 ✕로 회수 → ④ (선택) 일반 담당자가 된 기존 대표 삭제.
- ⚠️ 기존 대표가 **아무에게도 권한을 안 주고 떠나면 대표 기능이 잠깁니다**(DB 복구 필요). 로그인 이메일/`owner_email`(모델 메일 회신주소)도 함께 갱신 권장.

### 자주 묻는 것 / 주의점
- **"매니저가 매출이 안 보인대요"** → 정상. 담당자에서 그 사람의 `can_view_finance`를 켜주세요.
- **"대대행 발송 저장이 실패해요"** → DB 마이그레이션 `supabase/talent_distribution_v4_subagency.sql` 미적용일 수 있습니다. 코드 배포 전 적용 필요.
- **메뉴는 삭제하지 마세요** — 포트폴리오·패키지·발송 등은 권한/요금제에 따라 노출만 달라집니다.

### 이 역할이 보는 범위(권한)
- **전부 봅니다.** 담당자·회사설정·소유권 이전·재무·세무 식별번호까지 전권. 유일한 제약은 "대표 최소 1명" 안전장치.

## 2. 이매니저 — 섭외 담당 · member

> **한 줄 소개**: 모델과 고객을 관리하고, 들어온 문의를 섭외로 굴려 확정·일정까지 책임지는 현장 운영자.

### 자주 쓰는 메뉴
**섭외** · **모델** · **포트폴리오** · **패키지** · **고객사** · **대시보드** · (필요 시) **발송**.
> 재무 권한이 없으면 **매출·정산·세무 식별번호는 안 보입니다(정상).** 대표에게 `can_view_finance`를 요청하세요.

### 대표적인 하루 워크플로우
1. **대시보드 확인**: 상단 통계 + **신규 문의 카드**(상태=문의접수). 오늘 건은 **NEW**, 묵은 건은 **N일 경과**(3일↑ 빨강) — 오래된 것부터 처리.
2. **모델 등록·정비**(모델): 이름·생년월일 6자리·성별·타입·신체·연락처. **외국인 모델**은 상단 토글 → [비자·정산 정보] 모달. 3사이즈는 inch 우선(저장은 cm).
   - ⚠️ **세무 신고용 정보/세무 식별번호 입력란이 안 보이면** 재무 권한이 없는 것 — 정상입니다(주민번호 등은 권한자만 입력).
3. **포트폴리오 정리**(포트폴리오): 사진 최대 30장 드래그 정렬, 대표 지정, **컴카드** 생성. **영상(유튜브·비메오·인스타·틱톡) 링크 첨부**(9:16 세로형 지원, 클릭 시 라이트박스 재생) — 사진과 별개 항목.
   - 처음이라면 좌상단 **"썸네일 생성"** 버튼이 보일 때 1회 실행(로딩·데이터 절감).
4. **패키지(모델 제안)**: 정보 완비 모델 선택 → **패키징** → 공유 링크 자동 생성·복사 / PDF 발송. 각 모델 항목에 영상도 첨부 가능.
5. **섭외 추가/진행**(섭외): 타입(촬영/실물미팅/피팅/오디션) + 모델·고객사·담당자·일시·장소·금액. 카톡 등으로 온 문의는 상태 **문의접수**로 등록 → 대시보드 신규 문의에 노출.
   - 상태 흐름(촬영): 문의접수 → 모델제안 → 선택대기 → 스케줄확인 → (HOLD) → 섭외확정 → 촬영완료 → 정산완료/취소. 비촬영은 **요청 → 확정 → 완료**로 단순.
6. **확정 → 캘린더(수락형)**: 촬영·실물미팅을 **확정**하면 모델에게 **[수락]/[거절] 초대 메일**만 발송되고 **이 시점엔 캘린더 일정이 안 만들어집니다.** 모델이 **[수락]** 을 누르면 → 에이전시 구글 캘린더 생성 + 모델은 수락 페이지에서 .ics·구글추가.
   - **급할 땐** 섭외 상세 **"일정 보내기 → 구글 캘린더 등록"**(수동 override)으로 즉시 등록.
7. **알림톡 통보**(Standard 이상): 확정/취소/전날 리마인드/변경 4종. 담당자 연락처 = 멤버 `phone` → 에이전시 `rep_phone` 순.

### HOLD/충돌 대응 (순서 의존 규칙)
같은 모델·같은 날이라도 **시간이 실제로 겹치거나 버퍼 미만일 때만** HOLD. **먼저 시작하는 일정** 기준:
- 미팅·피팅·오디션끼리 **같은 장소** → 겹쳐도 **HOLD 안 함**.
- 미팅·피팅·오디션끼리 **다른 장소** → **1시간**.
- **앞=미팅류 → 뒤=촬영** → **1시간**(예: 11~12 미팅 → 13시 촬영 OK).
- **앞=촬영 → 뒤(촬영·미팅 무관)** → **2시간**.
- 충돌 시 **촬영 우선**, 미팅류가 **HOLD**로. 시간/장소를 충분히 벌리면 **자동 HOLD 해제**(→ CHECKING).
- 모델 휴무(`model_offs`)와 겹치면 **경고 후 강행 허용**(캘린더에 음영/"휴무 N").

### 자주 묻는 것 / 주의점
- **"확정했는데 캘린더에 일정이 없어요"** → 정상입니다. **모델이 수락해야** 생성됩니다(수락형). 급하면 수동 "구글 캘린더 등록".
- **"비구글(네이버·카카오) 모델은?"** → 수락 메일에 **구독 안내**가 자동 동봉됩니다. 한 번 **구독**하면 이후 변경이 자동 반영(§MANUAL 6-2).
- **"초대 메일이 안 가요"** → 대개 코드가 아니라 **Resend 도메인 인증/발신주소 외부 설정** 문제. 대표에게 확인 요청.
- **미수락 상태에서 일시·장소를 바꾸면** 초대 메일이 자동 재발송됩니다(중복 일정 방지 설계).

### 이 역할이 안 보는 것 (권한)
- 기본 member는 **매출·정산·세무 식별번호·세무사 엑셀**이 숨겨집니다. **담당자·회사설정·소유권**도 대표 전용이라 보이지 않습니다.
- 재무 열람이 필요하면 대표에게 `can_view_finance` 부여 요청 → 박정산 역할로 확장(아래 ③).

## 3. 박정산 — 재무 담당 · can_view_finance member

> **한 줄 소개**: 모델 지급·고객 입금·세무 서류를 맞추는 사람. 대표가 `can_view_finance`를 켜줘야 이 화면들이 보입니다.

### 자주 쓰는 메뉴
**정산** · **매출 현황** · **매출 거래명세서/청구** · (참고용) 모델 · 섭외.

### 대표적인 주간 워크플로우
1. **정산 탭으로 진행 상황 파악**: **정산대기 / 정산완료 / 미입금잔금** 분류 확인(분류 조건은 [김대표 §정산 승인](#정산-승인확인-관점) 참고). 필터: 날짜/모델/고객사.
2. **모델 정산액 산출**: 모델료(Day 9h / Half 5h / Hour 1h) + 정산방식(비율 % / 정액). 세무유형으로 자동 계산:
   - **프리랜서 3.3%**, **외국인(비자율)**, **소속사(대대행) +10% 계산서**.
3. **모델 발급 서류**(단일 모델 선택 시 — 세무유형 자동 분기):
   - **프리랜서·외국인 → 원천징수 내역서**(식별번호 라벨: 내국인=주민등록번호 / 외국인=여권·외국인등록번호).
   - **소속사(대대행) → 거래명세서**(공급가액 + 부가세 10% + 합계). 소속사는 **원천징수 대상 아님**(세액 0), **세금계산서는 소속 에이전시가 발행.**
   - 미리보기 → 🖨 인쇄/PDF → 📧 이메일.
4. **고객 입금 맞추기**(매출): 섭외 상세의 **계약금/잔금 입금 체크(또는 완납)** 가 입금 상태의 기준. 이걸 채워야 정산완료로 넘어갑니다.
5. **매출 현황 점검**: 모델/고객사 순위 → 필터. **매출총이익 = 매출(공급가) − 모델 정산 기준액**. **엑셀(.xlsx)** 다운로드(계약금·잔금·모델지급·총이익·합계).
   - ⚠️ 매출 인정 = **확정/완료/정산 상태 + 계약형 + 금액>0 + 촬영일이 선택 기간 내**. **미래월 건은 그 달을 봐야 보입니다(버그 아님).**
6. **고객사 거래명세서/청구**: 매출 화면 상단 **`📑 거래명세서/청구`** → 고객사 대상(공급자=당사).
   - **발급 단위 토글**: 프로젝트별 / 고객사 월합산.
   - 상태: **완납/부분입금/미수** + 합계카드 + `미수만 보기` 필터. 행의 **`🧾 명세서 발급`** → 공급가+VAT10%+합계+입금현황. ⚠️ 실제 세금계산서는 홈택스 별도 발행.

### 세무 식별번호는 어디서?
- **내국인 = 모델 폼 '세무 신고용 정보'**(주민등록번호 — 입력 시 생년월일 6자리 자동 채움).
- **외국인 = [비자·정산 정보] 모달**(주소 + 외국인등록번호/여권번호).
- 둘 다 **암호화 저장**(화면은 마스킹), `can_view_finance` 전용. 세무사 엑셀 다운로드는 **감사 로그**가 남습니다.

### 자주 묻는 것 / 주의점
- **"소속사인데 원천징수세액이 0이에요"** → 정상입니다(소속사는 원천징수 대상 아님, 거래명세서로 발급).
- **"이 매출이 합계에 안 잡혀요"** → 상태/계약형/금액/기간(촬영일) 4조건을 확인. 미래월은 해당 월 조회.
- **계산식은 임의로 바꾸지 마세요** — `modelPayout`(소속사=기준액×1.1), `clientCharge`(공급가×1.1), `modelWithholding`(소속사=0)은 이미 정확합니다.

### 이 역할이 안 보는 것 (권한)
- 여전히 **member**이므로 **담당자 관리·회사설정·소유권 이전**은 대표 전용 → 보이지 않습니다.
- 재무 권한이 **꺼지면** 위 화면 전부 다시 숨겨집니다(대표가 토글 제어).

## 4. (시나리오) 대대행 발송 — A 업체 ↔ B 업체

> **한 줄 소개**: A 업체가 자기 모델을 B 업체에 보내고, B가 그 모델을 **자기 모델로 편입**해 영업·정산까지 하는 흐름. 편입되는 순간 모델은 **대대행(소속사 고정)** 이 됩니다.

> ⚠️ **선행 조건(공통)**: DB 마이그레이션 `supabase/talent_distribution_v4_subagency.sql` 적용. 미적용 시 발송·편입·회사설정 저장이 실패합니다.

### A. A 업체 (모델을 보내는 쪽)

**자주 쓰는 메뉴**: 설정/회사정보 · 모델 · **발송**.

**워크플로우**
1. **회사 정산정보 1회 등록**(회사설정): 상호·사업자번호·대표·주소 + **정산 입금계좌(`agencies.payout_bank_info`)**. 이 정보가 발송 스냅샷의 출처가 됩니다.
2. **발송(SendTab)에서 모델 전송**: 발송 시 위 회사정보가 **발송 스냅샷 `talent_distributions.sender_payout_info`**(jsonb: company_name·biz_no·rep_name·contact·address·bank)에 실려 B에게 자동 전달됩니다 — A는 모델별로 업체정보를 다시 입력할 필요 없음.
3. **정산 관점**: A의 정산 화면은 기존 소속사 계산을 재사용해 **"A 지급액(○○ 대대행)"** 으로 표기됩니다. (집행은 수동 — 신규 계산식 없음.)

**A 입장 주의점**
- 회사 정산정보를 **한 번도 등록 안 하면** 발송 스냅샷이 비어 B의 업체정보 자동입력이 실패합니다 → **꼭 1회 등록**.
- 발송 후에는 B가 받은 모델을 **'내 모델로 등록'** 해야 편입이 완료됩니다(아래 B).

### B. B 업체 (모델을 받는 쪽)

**자주 쓰는 메뉴**: **발송**(받은 모델) · 모델 · 정산.

**워크플로우**
1. **'내 모델로 등록'**: A가 보낸 모델을 받아 등록하면 **대대행(소속사 `payout_tax_type="company"`) 고정** 으로 편입됩니다.
   - ⚠️ 이 모델은 **프리랜서/외국인으로 변경 불가**, **외국인 토글도 숨김**.
   - **A 업체정보(상호·사업자·대표·연락처·계좌)는 발송 스냅샷에서 자동 입력** — B가 손으로 칠 필요 없음.
2. **마진만 입력**: B는 모델별 **공급가(`fee_*`)와 기준액(`payout_*_value`)** 만 입력합니다(=B의 마진 설계).
3. **출처 자동 기록**: `models.source_agency_id / source_agency_name / source_distribution_id`에 자동 기록 → 모델 목록의 **'출처' 필터** + **'대대행' 배지**로 식별.
4. **정산**: 기존 company 경로(`modelPayout = 기준액 × 1.1`)를 그대로 재사용해 **"A 지급액(○○ 대대행)"** 으로 표시됩니다. 서류는 **거래명세서**(공급가+VAT10%+합계), 원천징수 0.

**B 입장 주의점**
- 편입 모델의 세무유형을 바꾸려 하지 마세요(설계상 **소속사 고정**).
- 계산식을 새로 만들지 마세요 — **기존 company 경로 재사용**이 원칙입니다.
- A 업체정보가 비어 보이면 → A가 회사 정산정보를 등록하지 않았거나, V4 SQL 미적용일 수 있습니다.

### 이 시나리오에서 안 보이는 것 / 권한
- 발송·편입·회사 정산정보 저장은 해당 업체의 **대표/권한자** 기준으로 동작합니다. 정산 표기·서류는 **재무 권한(`can_view_finance`)** 이 있어야 보입니다.
- 편입 모델의 **세무유형 변경 UI·외국인 토글**은 의도적으로 숨겨집니다(대대행 고정).
