# modiq 시작하기 — 단계별(Step-by-Step) 셋업 매뉴얼

> 처음 modiq를 도입하는 에이전시가 **가입부터 매출 관리까지** 순서대로 따라 하기 위한 안내서입니다.
> 각 단계는 같은 틀(목적 → 위치 → 할 일 → 권한/주의 → 완료 기준)로 정리되어 있어, 위에서부터 차례로 체크하며 진행하면 됩니다.
> **버전 v1.3.0 · 갱신일 2026-06-28 · 대상: 처음 셋업하는 에이전시(대표/매니저)**

---

## 셋업 한눈에 보기 (8단계 체크리스트)

| 순서 | 단계 | 메뉴 | 권한 | 예상 소요 |
|---|---|---|---|---|
| 0 | 가입 / 로그인 | (랜딩) | — | 3분 |
| ① | 회사정보 입력 | 설정(회사정보) | 대표 | 10분 |
| ② | 담당자 추가 | 담당자 | 대표 | 5분 |
| ③ | 모델 등록 | 모델 | 대표·매니저 | 모델당 3~5분 |
| ④ | 고객(고객사) 등록 | 고객사 | 대표·매니저 | 고객사당 2분 |
| ⑤ | 섭외 등록 | 섭외 | 대표·매니저 | 건당 3분 |
| ⑥ | 발송(대대행) *선택* | 발송 | 대표·매니저 | 5분 |
| ⑦ | 정산 | 정산 | 재무 권한 | 건당 2분 |
| ⑧ | 매출 확인 | 매출 현황 | 재무 권한 | 5분 |

> **전체 첫 셋업(①~⑤) 약 30분.** 모델·고객 데이터 양에 따라 ③·④가 늘어납니다.
> ⑥ 발송은 **대대행(다른 업체 모델을 주고받는 경우)에만** 필요한 선택 단계입니다.

**진행 체크리스트(복사해서 사용)**
- [ ] 0. 구글 계정으로 가입·로그인, 에이전시 생성(자동 대표)
- [ ] ① 회사정보: 상호·대표자·연락처·사업자번호·주소·정산 입금계좌·로고
- [ ] ② 담당자 추가 + 각자 연락처, 필요 시 재무 열람 권한
- [ ] ③ 모델 등록(개별 또는 엑셀 일괄), 사진은 포트폴리오에서
- [ ] ④ 고객사 등록(담당자·연락처·계산서 이메일)
- [ ] ⑤ 섭외 등록(타입·모델·고객사·일시·금액)
- [ ] ⑥ (대대행 시) 파트너 연결 → 발송 / 받은 모델 편입
- [ ] ⑦ 정산: 모델 지급·서류 발급
- [ ] ⑧ 매출: 기간별 매출·미수금 확인

## 0. 가입 / 로그인

- **이 단계의 목적**: modiq 계정을 만들고 내 에이전시를 생성합니다.
- **위치**: 서비스 첫 화면(랜딩)
- **할 일**:
  1. **회사 대표(운영) 구글 계정**으로 로그인합니다. (캘린더 연동·모델 메일 회신주소가 이 계정 기준으로 잡힙니다)
  2. 처음 로그인하면 에이전시를 생성하고, 만든 사람이 자동으로 **대표(owner)** 가 됩니다.
  3. 신규 가입은 **14일 무료 체험(trial)** 으로 시작합니다(전 기능 사용 가능).
- **권한/주의**:
  - 로그인 이메일과 `owner_email`(모델 메일 **회신주소**)은 **최초 생성자 기준**으로 남습니다. 나중에 대표가 바뀌면 회신주소도 함께 변경하세요.
  - 가입은 **반드시 운영용 구글 계정**으로. 개인 계정으로 만들면 캘린더 동기화가 그 계정에 묶입니다.
- **✅ 완료 기준**: 로그인 후 대시보드가 보이고, 좌측(모바일은 하단/더보기) 메뉴가 나타난다.
- **다음 단계 →** ① 회사정보 입력

---

## ① 회사정보 입력 (설정)

- **이 단계의 목적**: 명세서·알림톡·발송에 쓰일 우리 회사 기본정보를 1회 등록합니다.
- **위치**: **설정(회사정보)** 메뉴 — *대표만 보입니다.*
- **할 일** (입력 항목):
  1. **상호 (에이전시명)** — 알림톡 **발신명**으로도 사용됩니다.
  2. **대표자명**
  3. **대표 연락처**(`rep_phone`) — 알림톡 담당자 연락처 **fallback**.
  4. **알림톡 문의 연락처** — 모델·고객사에게 표시되는 대표 문의번호.
  5. **사업자등록번호** — 10자리 체크섬 검증(잘못되면 저장 거부).
  6. **주소** — `주소 검색` 버튼으로 우편번호 검색 후 상세주소 입력.
  7. **정산 입금계좌** — 모델 발송(대대행) 시 상대에게 전달되는 은행/계좌/예금주. (예: `국민 123456-78-901234 (주)아이모`)
  8. **계산서 발송 이메일**(`payout_tax_email`) — 세무 관련 수신 메일.
  9. **회사 로고** — 명세서 좌측 상단에 표시(투명 PNG 권장).
- **권한/주의**:
  - 이 메뉴는 **대표 전용**입니다. 매니저(member)에게는 보이지 않습니다(정상).
  - **정산 입금계좌**는 ⑥ 발송(대대행)에서 받는 업체에게 전달되므로, 대대행 발신을 한다면 **반드시** 채워두세요. 비워두면 발송 스냅샷에 빈 값이 실립니다.
  - 같은 화면 아래 **"권한 부여(공동 대표)"** 카드에서 공동 대표 지정/회수가 가능합니다(②와 연계).
- **✅ 완료 기준**: 상호·사업자번호·주소·정산 입금계좌가 모두 저장됨(우측 미리보기 카드에 반영).
- **다음 단계 →** ② 담당자 추가

---

## ② 담당자 추가 (담당자)

- **이 단계의 목적**: 함께 운영할 매니저를 등록하고 권한을 정합니다.
- **위치**: **담당자** 메뉴 — *대표만 보입니다.*
- **할 일**:
  1. `담당자 추가`로 매니저를 등록합니다(이름·이메일).
  2. 각 담당자의 **연락처(phone)** 를 채웁니다 → 알림톡 담당자 연락처로 **우선** 사용(없으면 회사 `rep_phone`).
  3. 재무(매출·정산)를 봐야 하는 담당자는 **재무 열람 권한(`can_view_finance`) 토글을 ON** 합니다.
- **권한/주의**:

  | 권한 | 보이는 것 |
  |---|---|
  | **대표(owner)** | 전체(설정·담당자·매출·정산·소유권 이전) |
  | **담당자(member)** | 기본은 **매출·정산 숨김** |
  | **담당자 + 재무 열람 ON** | 매출·정산·세무 식별번호까지 열람 |

  - 권한 공식: `canViewFinance = 대표 || (담당자 && 재무 열람 ON)`.
  - **대표(owner)는 담당자 화면에서 바로 삭제되지 않습니다.** 먼저 설정의 "권한 회수"로 일반 담당자로 내린 뒤 삭제 버튼이 생깁니다.
  - **대표는 최소 1명 유지**(마지막 대표는 회수 불가 — 계정 잠김 방지). 본인 권한은 회수 불가.
  - 담당자 수는 요금제 한도를 따릅니다(초과 시 인당 추가요금, ⑨ 요금제 참고).
- **✅ 완료 기준**: 매니저가 목록에 보이고, 재무 담당자는 권한 ON 상태.
- **다음 단계 →** ③ 모델 등록

---

## ③ 모델 등록 (모델)

- **이 단계의 목적**: 소속 모델을 등록해 섭외·정산의 기준 데이터를 만듭니다.
- **위치**: **모델** 메뉴 (사진은 **포트폴리오** 메뉴에서)
- **할 일**:
  1. `모델 추가`(모달 폭 680px)에서 입력: 이름 · **생년월일 6자리(YYMMDD)** · 성별 · 모델 타입 · 신체 · 활동분야 · 연락처.
  2. **3사이즈(가슴·허리·엉덩이)**: inch 우선 입력(기본 inch). cm로 넣으면 inch로 자동 변환 표기됩니다.
  3. **외국인 모델**: 상단 **"외국인 모델" 토글** → **[비자·정산 정보]** 모달에서 비자 유형·입출국일·세율·지급방식 + **주소·외국인등록번호/여권번호**를 한 화면에서 입력(세무유형 자동 반영).
  4. **세무 신고용 정보**: 내국인=주민등록번호(입력 시 생년월일 6자리 자동 채움) / 외국인=식별번호.
  5. 사진은 **포트폴리오** 메뉴에서: 모델 선택 → 드래그앤드롭/＋로 추가(**최대 30장**), 대표 사진 지정, 컴카드 생성.
  6. **일괄 등록**: `일괄 등록` 버튼으로 엑셀 업로드.
- **권한/주의**:
  - **세무 신고용 정보·세무 식별번호는 재무 권한 계정에만 보입니다**(암호화 저장·마스킹 표시). 권한 없는 매니저에게 숨겨지는 것이 정상입니다.
  - 포트폴리오에 **"썸네일 생성"** 버튼이 보이면 1회 실행하세요(목록·갤러리 로딩과 데이터 사용량 대폭 감소).
  - 사진 저장 시 자동으로 긴 변 1500px로 정리되고 480/150px 썸네일이 생성됩니다.
  - 범위 필터(키·나이 등)에서 **값이 비어 있는 모델은 숨겨지지 않습니다**(누락 방지).
- **✅ 완료 기준**: 모델이 목록에 보이고, 대표 사진 썸네일이 카드에 표시됨.
- **다음 단계 →** ④ 고객(고객사) 등록

---

## ④ 고객(고객사) 등록 (고객사)

- **이 단계의 목적**: 섭외·매출·명세서를 연결할 고객사를 등록합니다.
- **위치**: **고객사** 메뉴
- **할 일**:
  1. `고객사 추가`(모달 폭 680px)에서 입력: 회사명 · 담당자 · 연락처 · 이메일.
  2. **계산서 발송 이메일**을 채워두면, ⑧ 매출의 거래명세서/청구 발송 시 **우선 수신 주소**로 사용됩니다.
- **권한/주의**:
  - 고객사는 섭외·매출의 연결 키입니다. 같은 고객을 중복 등록하지 않도록 등록 전 검색해보세요.
  - 프로젝트 단위 명세서를 쓰려면, ⑤ 섭외에서 **같은 프로젝트명**으로 묶어야 ⑧에서 한 장으로 발급됩니다.
- **✅ 완료 기준**: 고객사가 목록에 보이고, 계산서 이메일이 입력됨.
- **다음 단계 →** ⑤ 섭외 등록

---

## ⑤ 섭외 등록 (섭외)

- **이 단계의 목적**: 실제 일(촬영/미팅 등)을 등록하고 일정·금액을 관리합니다.
- **위치**: **섭외** 메뉴 (모바일 하단 탭)
- **할 일**:
  1. `섭외 추가`(단일/프로젝트, 모달 폭 680px) → **타입 선택**: **촬영 / 실물미팅 / 피팅 / 오디션**.
  2. 모델 · 고객사 · **담당자(멤버 드롭다운)** · 일시 · 장소 · 금액 입력.
  3. 카카오/채팅으로 들어온 문의는 상태 **문의접수(INQUIRY)** 로 등록 → 대시보드 **신규 문의 카드**에 노출됩니다.
  4. **확정(CONFIRMED)** 으로 전환하면 모델에게 **[수락]/[거절] 초대 메일**이 자동 발송됩니다(캘린더는 모델 수락 후 생성 — ⑩ 참고).
- **권한/주의**:
  - **상태 흐름** — 촬영: 문의접수 → 모델제안 → 선택대기 → 스케줄확인 → (HOLD) → 섭외확정 → 촬영완료 → 정산완료 / 취소. 비촬영(미팅·피팅·오디션): 요청 → 확정 → 완료(+취소).
  - **스케줄 충돌/자동 HOLD**(같은 모델·같은 날, 시간이 겹치거나 간격 부족일 때만):
    - 미팅류끼리 **같은 장소** → 겹쳐도 HOLD 안 함 / **다른 장소** → 1시간.
    - 앞=미팅류 → 뒤=촬영 → 1시간. 앞=촬영 → 뒤 → 2시간.
    - 충돌 시 **촬영 우선**, 미팅류가 HOLD로 내려갑니다(간격 벌리면 자동 해제).
  - **휴무 충돌**: 모델 기간 휴무와 겹치면 경고 후 강행 허용.
  - ⚠️ **계약형(hasContract)·금액>0 인 섭외만** 매출로 인정됩니다(⑧).
  - 초대 메일이 실제 발송되려면 **Resend 도메인 인증**이 되어 있어야 합니다(외부 설정).
- **✅ 완료 기준**: 섭외가 목록·캘린더에 보이고, 확정 건은 모델 수락 배지가 표시됨.
- **다음 단계 →** ⑥ 발송(선택) 또는 ⑦ 정산

---

## ⑥ 발송 (선택 · 대대행) (발송)

- **이 단계의 목적**: 다른 에이전시와 **모델 자료를 단방향으로 주고받습니다**(대대행).
- **위치**: **발송** 메뉴 (탭: **파트너 / 보내기 / 보낸함 / 받은함**)
- **할 일**:

  **A 역할(모델을 보내는 쪽)**
  1. **파트너** 탭 → 상대 사업자번호로 **파트너 신청** → 상대가 수락하면 연결됩니다.
  2. **보내기** 탭(3단계 마법사): ① 모델 선택 → ② 파트너 지정 → ③ 미리보기·발송. (발송 메모 선택)
  3. 발송 시 **회사설정의 업체정보(상호·사업자·대표·주소·정산 입금계좌)** 가 스냅샷으로 함께 전달됩니다 → ①에서 미리 등록해 두세요.

  **B 역할(모델을 받는 쪽)**
  1. **받은함** 탭에서 받은 모델을 확인하고, 카드 하단 **"내 모델로 등록"** 을 누르면 내 모델 목록으로 **편입**됩니다.
  2. 편입 후에는 **모델** 메뉴에서 **마진(공급가 `fee_*` · 기준액)만** 입력하면 됩니다.
- **권한/주의**:
  - ⚠️ **정산가·세무정보·연락처·주민번호는 발송되지 않습니다.** 노출가(Day/Half/Hour)·신체정보·사진만 전달됩니다.
  - 받아서 편입한 모델은 **대대행(소속사, 세무 10% 고정)** 으로 고정됩니다 — 프리랜서/외국인 변경 불가, 업체정보 자동 입력, 모델 목록에 **'대대행' 배지** + **출처** 필터.
  - 정산 시 기존 소속사 계산(`기준액×1.1`)을 재사용해 **"A 지급액(○○ 대대행)"** 으로 표기됩니다(집행은 수동).
  - 보낸함에서 **철회**하면 상대 수신함에서 사라집니다.
- **✅ 완료 기준**: (A) 보낸함에 발송 건이 보임 / (B) 받은 모델이 '대대행' 배지로 내 모델 목록에 보임.
- **다음 단계 →** ⑦ 정산

## ⑦ 정산 (정산)

- **이 단계의 목적**: 촬영 완료된 건의 모델 지급액을 산출하고 서류를 발급합니다.
- **위치**: **정산** 메뉴 — *재무 권한(대표 또는 재무 열람 ON) 전용.*
- **할 일**:
  1. 필터: **날짜 / 모델 / 고객사**로 건별 리스트 조회.
  2. 모델료(Day 9h / Half 5h / Hour 1h) + 정산방식(비율 % / 정액)으로 모델 정산액 확인.
  3. **모델 발급 서류**(단일 모델 선택 시 세무유형 자동 분기):
     - **프리랜서·외국인 → 원천징수 내역서** (식별번호: 내국인=주민등록번호 / 외국인=여권·외국인등록번호)
     - **소속사(대대행) → 거래명세서** (공급가액 + 부가세 10% + 합계). 소속사는 원천징수 0, 세금계산서는 소속 에이전시가 발행.
  4. 미리보기 → 🖨 인쇄/PDF → 📧 이메일 발송.
- **권한/주의**:
  - **정산 탭 분류**: 정산대기=촬영완료 & (고객입금·모델지급 둘 다 완료 아님) / 정산완료=둘 다 완료(또는 SETTLED) / 미입금잔금=고객 미입금.
  - 정산 헤더 라벨은 **센터 정렬**(리스트 정렬 규칙의 예외).
  - **세무사용(주민번호 포함) 엑셀**은 재무 권한만 받을 수 있고, 다운로드 시 **보안 감사 로그**가 남습니다.
  - 정산 계산식(소속사=기준액×1.1, 원천징수=0 등)은 **임의 수정 금지**(이미 정확).
- **✅ 완료 기준**: 모델 지급 완료 처리, 필요 서류 발급(원천징수/거래명세서).
- **다음 단계 →** ⑧ 매출 확인

## ⑧ 매출 확인 (매출 현황)

- **이 단계의 목적**: 기간별 매출·총이익·미수금을 보고, 고객사 거래명세서를 발급합니다.
- **위치**: **매출 현황** 메뉴 — *재무 권한 전용.*
- **할 일**:
  1. 기간을 선택하고 순위에서 모델/고객사 클릭 → 해당 매출만 필터(요약 카드 연동).
  2. **매출총이익 = 매출(공급가) − 모델 정산 기준액** 카드 확인.
  3. **엑셀(.xlsx) 다운로드**(섭외ID·계약금·잔금·모델지급·총이익·합계).
  4. **거래명세서/청구**(상단 `📑 거래명세서 / 청구`): 고객사 대상, **프로젝트별** 또는 **고객사 월합산** 토글로 발급 → 행의 `🧾 명세서 발급` → 인쇄/PDF/이메일(고객사 계산서 이메일 우선).
- **권한/주의**:
  - **매출 인정 필터**: 확정·완료·정산 상태 + 계약형 + 금액>0 + **촬영일이 선택 기간 내**. 미래월 건은 **그 월/기간을 봐야** 보입니다(버그 아님).
  - 입금 상태(완납/부분입금/미수)는 섭외 상세의 **계약금/잔금 입금 체크**를 기준으로 잡힙니다.
  - 계산식(`clientCharge=공급가×1.1`)은 **신규 생성·임의 수정 금지**. ⚠️ 실제 세금계산서는 홈택스에서 별도 발행.
- **✅ 완료 기준**: 기간 매출·미수금이 카드에 보이고, 필요 시 명세서가 발급됨.

# 보조 가이드 (셋업 이후 일상 운영)

## 대시보드 읽는 법

- **상단 통계 카드**: 진행중 섭외 · 이번 달 매출 등.
- **신규 문의 카드**: 상태가 **문의접수**인 건 모음. 오늘 건은 **NEW**, 묵은 건은 **N일 경과**(3일↑ 빨강). 행 클릭 → 섭외 상세에서 확정/취소.
  - ※ 문의접수 건은 "진행중 섭외" 통계에서 **제외**(중복 방지).
- **이번 주 섭외**: 날짜 클릭 시 캘린더로 이동.
- 진행중 섭외·신규 문의 목록은 섭외 화면과 **같은 공용 컴포넌트**로 그려져 어디서 봐도 컬럼 정렬이 동일합니다.

## 포트폴리오 · 패키지

- **포트폴리오**: 모델별 사진 갤러리·대표 지정·컴카드(A4 가로) 생성. **영상(유튜브·비메오·인스타·틱톡, 9:16 지원)** 임베드 가능.
- **패키지(모델 제안)**: 정보 완비 모델(이름+사진+키)을 선택 → 패키징 → **공유 링크 자동 생성**. 공개 페이지는 대표 1컷 카드 + 클릭 시 갤러리, 신체정보 자동 반영. 패키지 항목에도 영상 첨부 가능.
- ⚠️ 두 메뉴는 데스크탑·모바일 더보기에 모두 있으며 삭제 대상이 아닙니다.

## 캘린더 · 알림톡

- **수락형 캘린더(기본)**: 섭외 **확정** → 모델에 [수락/거절] 초대 메일 → 모델 수락 시 에이전시 구글 캘린더에 일정 생성 + 모델은 .ics/구글추가. 즉시 넣으려면 섭외 상세 **"일정 보내기 → 구글 캘린더 등록"**(수동).
  - 구글 외 이메일(네이버·카카오 등) 모델은 수락 메일에 **구독 링크**가 자동 동봉됩니다.
- **구독형(권장)**: 모델 상세 **"구독 링크"** 로 안내 페이지 전달 → 모델이 최초 1회 구독하면 이후 변경·취소 자동 반영(아이폰=캘린더 구독 / 안드로이드=ICSx⁵ / PC=URL로 추가). 동기화는 주기적(실시간 아님) — 급한 변경은 알림톡/이메일로 별도 통보.
- **알림톡(Pro 이상)**: 확정/취소/전날 리마인드/변경 4종. 담당자 연락처 우선순위 = 멤버 phone → 회사 rep_phone. 외부 준비(카카오 채널·템플릿 승인 → `solapi-send` 배포 → 환경변수) 전에는 발송이 안전하게 no-op.
- ⚠️ 메일·알림톡 미발송은 대개 **코드가 아니라 외부 설정**(Resend 도메인 인증, Solapi 채널) 문제입니다.
